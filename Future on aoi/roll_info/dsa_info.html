Understanding Complexity: Big O Notation

Big O notation is how we measure an algorithm's efficiency. It describes how the runtime or memory usage grows as the input size increases. Mastering this concept is crucial for comparing algorithms and making performance-conscious decisions.

Core Data Structures

Arrays (Lists in Python)

Analogy: A numbered list of storage boxes. Accessing any box is fast if you know its number, but adding a new box in the middle is slow.

Complexity: Access: O(1), Search: O(n), Insert/Delete: O(n).

# Python lists are dynamic arrays
my_list = [10, 20, 30]
print(my_list[1]) # Fast access: 20
my_list.append(40) # Add to end
my_list.insert(1, 15) # Slow insert

Linked Lists

Analogy: A treasure hunt where each clue points to the next. Fast to add/remove clues, but finding a specific one requires following the trail.

Complexity: Access: O(n), Search: O(n), Insert/Delete: O(1) (at head/tail).

# Basic node structure
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

Stacks

Analogy: A stack of plates. You can only add or remove from the top. (Last-In, First-Out - LIFO).

Complexity: Push/Pop/Peek: O(1).

# Python lists can act as stacks
stack = []
stack.append('A') # Push
stack.append('B')
print(stack.pop()) # Pop: returns 'B'

Queues

Analogy: A line at a checkout counter. The first person in line is the first one out. (First-In, First-Out - FIFO).

Complexity: Enqueue/Dequeue: O(1).

from collections import deque
queue = deque()
queue.append('A') # Enqueue
queue.append('B')
print(queue.popleft()) # Dequeue: 'A'

Hash Tables (Dicts)

Analogy: A dictionary. You look up a word (key) to instantly find its definition (value). Super fast lookups.

Complexity: Insert/Delete/Search: Average O(1).

# Python dictionaries are hash tables
my_dict = {'name': 'Alice', 'age': 30}
print(my_dict['name']) # Fast lookup
my_dict['city'] = 'New York'

Trees & Graphs

Analogy: A family tree (hierarchical) or a map of connected cities (network). Used for representing relationships.

Use Cases: File systems, social networks, network routing.

# Basic tree node
class TreeNode:
    def __init__(self, value):
        self.value = value
        self.left = None
        self.right = None

Essential Algorithms

Searching Algorithms

Linear Search: Checks each element one by one. O(n).

Binary Search: Efficiently finds an item in a *sorted* array by repeatedly dividing the search interval in half. O(log n).

Sorting Algorithms

Bubble/Selection/Insertion Sort: Simple but inefficient. O(nÂ²).

Merge Sort & Quick Sort: Efficient, divide-and-conquer algorithms. Average O(n log n).

Graph Traversal

Breadth-First Search (BFS): Explores neighbor nodes first (level by level). Used for finding the shortest path in an unweighted graph.

Depth-First Search (DFS): Explores as far as possible along each branch before backtracking. Used for cycle detection and topological sorting.
