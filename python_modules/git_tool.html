Git Commands: Your Local Toolkit

1. Setting Up a Repository

CommandDescriptionUse Case
git initInitializes a new, empty Git repository.Start tracking a brand new project.
git clone [url]Creates a local copy of a remote repository.Get a local working copy of a project.
git configConfigures Git settings (user, email, etc.).Set your identity for commits.

2. Staging & Committing Changes

CommandDescriptionUse Case
git add [file]Adds a file to the staging area.Prepare specific changes for commit.
git add .Adds all new and modified files to staging.Quickly stage all current changes.
git commit -m "[msg]"Records staged changes to the repository.Save a snapshot of your work.
git statusShows the status of your working directory.See what's changed or staged.

3. Branching & Merging

CommandDescriptionUse Case
git branchLists all local branches.See what branches exist locally.
git branch [name]Creates a new branch.Work on a new feature in isolation.
git checkout [name]Switches to the specified branch.Start working on a different branch.
git checkout -b [name]Creates a new branch and switches to it.A shortcut for creating and checking out.
git merge [name]Merges a branch into the current one.Integrate feature changes into main.
git branch -d [name]Deletes the specified local branch.Clean up unneeded branches.

4. Working with Remote Repositories

CommandDescriptionUse Case
git remote -vLists all remote repositories.Verify remote server connections.
git remote add [n] [url]Adds a new remote repository.Connect to another remote server.
git fetch [remote]Downloads changes from a remote.See remote changes without merging.
git pullFetches and merges remote changes.Update your local branch.
git push [rem] [br]Pushes changes to a remote repository.Share your work with others.

5. Inspecting & Undoing Changes

CommandDescriptionUse Case
git logShows the commit history.Review the project`s history.
git diffShows differences between commits, etc.See exact line-by-line changes.
git reset [file]Unstages a file, but preserves contents.Remove a file from the staging area.
git revert [commit]Creates a new commit that undoes another.Safely undo a public commit.
git stashTemporarily saves uncommitted changes.Switch branches without committing.
git stash popApplies the most recently stashed changes.Re-apply saved changes.

GitHub CLI: Power in Your Terminal

1. Repository Management

CommandDescriptionUse Case
gh repo createCreates a new repository on GitHub.Start a new project on GitHub.
gh repo clone [repo]Clones a repository from GitHub.Get a local copy of a GitHub repo.
gh repo viewViews a repository in the browser.Quickly open a repo`s GitHub page.

2. Pull Request Management

CommandDescriptionUse Case
gh pr listLists pull requests in a repository.See all open pull requests.
gh pr createCreates a new pull request.Propose changes for review.
gh pr checkout [pr]Checks out a pull request locally.Easily test a contributor`s changes.
gh pr mergeMerges a pull request.Merge an approved pull request.

3. Issue Management

CommandDescriptionUse Case
gh issue listLists issues in a repository.Find tasks to work on.
gh issue createCreates a new issue.Report a bug or request a feature.
gh issue statusShows the status of your issues.Track your reported issues.

This guide covers the most common commands. For advanced usage, refer to the official Git and GitHub documentation.
