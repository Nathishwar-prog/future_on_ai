The Critical Commands (80% of Use Cases)

Master these commands to handle the vast majority of your version control tasks.

1. Repository Setup

git init

Initializes a new Git repository in the current directory.

git init

Visualization:

Current Folder:
my_project/
↓
git init
↓
my_project/
.git/ (Git initialized!)

git clone

Clones an existing repository into a new directory.

git clone <repo_url>

Visualization:

GitHub/Remote
[Repo]
↓ Clone
Local Machine:
my_repo/
.git/
files...

git config

Gets and sets repository or global options (e.g., user info).

git config --global user.name "Your Name"

Visualization:

Git Settings:
User Name: "Not Set"
↓ config
User Name: "Your Name"
(Global or Local)

2. Basic Workflow

git status

Shows the working tree status, staged, modified, and untracked files.

git status

Visualization:

Working Dir:
file_A (Untracked)
file_B (Modified)
Staging Area:
file_C (Staged)

git add

Adds file contents to the index (staging area) for the next commit.

git add .

Visualization:

Working Dir:
file_A (Modified)
↓ add
Staging Area:
file_A (Staged)

git commit

Records the staged changes to the repository with a message.

git commit -m "Initial commit"

Visualization:

Staging Area:
file_A (Staged)
↓ commit
Local Repo:
(abc123) "Initial commit"
(HEAD -> main)

git log

Shows commit logs (history) of the repository.

git log --oneline

Visualization:

abc123 feat: Add header (HEAD -> main)
def456 fix: Fix button style
ghi789 docs: Update readme

3. Branching & Merging

git branch

Lists, creates, or deletes branches.

git branch feature/new-feat

Visualization (after creation):

abc123 Initial commit (HEAD -> main)
| \
| (feature/new-feat)

git checkout

Switches branches or restores working tree files.

git checkout feature/new-feat

Visualization (after switch):

abc123 Initial commit (main)
| \
| (HEAD -> feature/new-feat)

git merge

Joins two or more development histories together.

git merge feature/new-feat

Visualization:

A -- Commit on main
| \
B -- Commit on feature
| /
M -- Merge commit

Why Are These Critical?

Core Workflow: These commands form the backbone of daily Git usage, from initializing a project to committing changes and collaborating with others via remote repositories.

Foundation of Version Control: Mastering `init`, `add`, `commit`, `push`, `pull`, `branch`, `checkout`, and `merge` allows you to manage project history, work on features in isolation, and integrate changes effectively.

The Important Commands (Supporting Roles)

Frequently used to support critical commands and handle common specialized tasks.

1. History & Comparison

git diff

Shows changes between the working directory, staging area, and commits.

git diff <file>

Visualization:

File: my_file.txt
- Old Line
+ New Line
(Shows differences)

git fetch

Downloads objects and refs from another repository without merging.

git fetch origin

Visualization:

Remote: [C1]-[C2]-[C3]
↓ Fetch
Local: [C1]-[C2]
(origin/main points to C3, but local main is still at C2)

git tag

Creates, lists, deletes, or verifies a tag object (e.g., for releases).

git tag -a v1.0 -m "Release v1.0"

Visualization:

abc123 -- Commit A
def456 -- Commit B (v1.0)
ghi789 -- Commit C

2. Undoing Changes & Cleanup

git reset

Resets current HEAD to the specified state (undoes commits/stages).

git reset HEAD~1

Visualization (soft reset):

Local: C1 - C2 - C3 (HEAD)
↓ reset
Local: C1 - C2 (HEAD)
(C3 changes are now staged)

git revert

Creates a new commit that undoes the changes made by a previous commit.

git revert <commit_hash>

Visualization:

History: C1 - C2 (Bug) - C3
↓ revert C2
History: C1 - C2 - C3 - C4 (Revert C2)
(History preserved)

git stash

Temporarily shelves changes in a dirty working directory.

git stash save "WIP: feature X"

Visualization:

Working Dir: [Modified Files]
↓ stash
Stash List: [WIP: feature X]
Working Dir: [Clean]

3. History Rewriting & File Management

git rebase

Reapplies commits on top of another base tip for a cleaner history.

git rebase main

Visualization:

Old: A--B--C (main)
\
D--E (feature)
↓ rebase
New: A--B--C--D'--E' (main, feature)

git rm

Removes files from the working tree and from the index.

git rm old_file.txt

Visualization:

Working Dir: [file_A, old_file.txt]
↓ rm
Working Dir: [file_A]
Staging: (old_file.txt removed)

git mv

Moves or renames a file, a directory, or a symlink.

git mv old_name.txt new_name.txt

Visualization:

Working Dir: [old_name.txt]
↓ mv
Working Dir: [new_name.txt]
Staging: (renamed)

Why Are These Important?

Enhance Workflow: Commands like `diff` and `fetch` provide crucial insights and updates without altering your local work, enabling more informed decisions.

History Management: `reset`, `revert`, and `rebase` are powerful tools for correcting mistakes, cleaning up commit history, and maintaining a clear project timeline.

Context Switching: `stash` is invaluable for temporarily saving work when you need to switch contexts quickly, preventing unfinished changes from cluttering your branches.

The Utility Commands (Specialized Tools)

Valuable for edge cases, convenience, or specific advanced applications.

1. Diagnostics & Inspection

git reflog

Manages reflog information (records updates to the tip of branches).

git reflog

Visualization:

def456 HEAD@{1}: checkout: moving to main
ghi789 HEAD@{2}: commit: Initial commit
(Your Git "undo" history)

git blame

Shows what revision and author last modified each line of a file.

Visualization:

ghi789 (Alice 2023-01-10 11:45) function greet() {

git show

Shows various types of objects (e.g., commit, tag, tree, blob).

Visualization:

Commit: abc123
Author: Alice <alice@example.com>
Date: Mon Jan 1 10:00:00 2023

+ new_line

2. Advanced Workflow & Cleanup

git clean

Removes untracked files from the working tree.

git clean -df

Visualization:

Working Dir:
tracked_file.txt
untracked_log.txt
↓ clean
Working Dir:
tracked_file.txt

git bisect

Finds by binary search the commit that introduced a bug.

git bisect start

Visualization:

C1 (Good) --- C2 --- C3 --- C4 (Bad)
↓ bisect
Testing C2... (Good!)
Testing C3... (Bad!)
Bug introduced in C3!

git cherry-pick

Applies the changes introduced by some existing commits.

git cherry-pick <commit_hash>

Visualization:

Main: A -- B -- C
Feature: D -- E (Bugfix) -- F
↓ cherry-pick E on Main
Main: A -- B -- C -- E'
(Specific commit applied)

Why Are These "Utility"?

Problem Solving: Commands like `bisect` and `reflog` are indispensable for debugging, finding the origin of bugs, or recovering lost work.

Granular Control: `cherry-pick` allows for precise application of changes, while `clean` helps maintain a tidy working directory, especially in complex projects.

Deeper Insights: `blame` and `show` provide detailed information about file history and Git objects, useful for understanding code origins and repository internals.
